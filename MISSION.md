# Mission: Skribble Architecture Mastery

## Why
I want to fully understand every function and component in my Skribble codebase so I can confidently maintain, debug, extend, and explain the architecture to others. When I have this skill, I can make design changes without breaking things, onboard contributors, and build new features without consulting the codebase line-by-line.

## Success looks like
- I can trace any user action (login, create room, draw, guess) through every layer of the system
- I can explain the purpose of every server-side function and client-side component from memory
- I can identify which files to modify when adding a new feature (e.g., a new game event, a new page)
- I can diagnose where a bug lives by reasoning about the architecture before reading code

## Constraints
- Learning through interactive lessons tied to the actual codebase
- Prefer visual architecture diagrams and concrete code references over abstract explanations
- Lessons should be completable in under 10 minutes each

## Out of scope
- Writing new features (this is about understanding, not building)
- Deep dives into third-party library internals (Socket.IO, Mongoose, etc.) — only how they're used here
- Deployment and DevOps concerns
