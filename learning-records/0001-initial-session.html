Key Insights

  • The architecture is two-process (browser ↔ server) with Redis for state. No REST API — all game actions over Socket.IO.
  • The Room object is the central data structure — identical shape on client and server.
  • Server is source of truth. Client state is a projection of server events.
  • Several bugs were identified: undo sync issue, message context cleanup, timer leak, hardcoded IP.
  • Scoring: guessers get max(200 - timeToGuess, 0), drawer gets 50 + (guessers × 10).
