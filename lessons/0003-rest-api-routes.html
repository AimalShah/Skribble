Every HTTP endpoint: what it does, what it expects, what it returns.

Route Map

/api/auth POST /register → Create account, return JWT POST /login → Verify credentials, return JWT GET /me → Return current user (auth required) /api/rooms POST / → Create a room (auth required) GET /feed → List published rooms (public) POST /:roomId/publish → Publish to feed (owner only) POST /:roomId/unpublish → Remove from feed (owner only) /api/profile GET /:userId → View any user's profile (public) PUT / → Update own profile (auth required) /api/stats POST /update → Increment game stats (auth required) /api/health GET / → { status: "ok" }

Auth Middleware

Before diving into routes, understand the guard. server/src/middleware/auth.ts does three things:

authMiddleware(req, res, next) {
  // 1. Extract "Bearer <token>" from Authorization header
  // 2. Verify JWT with jwt.verify(token, JWT_SECRET)
  // 3. Attach decoded userId to req.userId
  //    → If invalid: return 401
  //    → If valid: call next()
}

signToken(userId) {
  return jwt.sign({ userId }, JWT_SECRET, { expiresIn: "7d" });
}

Every route marked "auth required" runs this middleware before its handler.

Auth Routes

server/src/routes/auth.ts

POST /api/auth/register

// Request body:
{ email: string, password: string, displayName: string }

// Logic:
1. Validate all fields present
2. Check if email already exists → 409 Conflict if yes
3. Create User document (password auto-hashed by pre-save hook)
4. Generate JWT token
5. Return { token, user: { email, displayName, avatar, stats } }

POST /api/auth/login

// Request body:
{ email: string, password: string }

// Logic:
1. Find user by email (select: "+password" to include hash)
2. If not found → 401
3. Compare password with bcrypt
4. If mismatch → 401
5. Generate JWT token
6. Return { token, user: { email, displayName, avatar, stats } }

GET /api/auth/me

// Headers: Authorization: Bearer <jwt>

// Logic:
1. authMiddleware extracts userId from JWT
2. Find user by userId (no password field)
3. Return { user }

Room Routes

server/src/routes/rooms.ts

POST /api/rooms

// Headers: Authorization: Bearer <jwt>
// Request body:
{
  name: string,
  isPrivate: boolean,
  password?: string,      // required if isPrivate
  language: string,
  settings: Settings
}

// Logic:
1. Generate UUID-style roomId
2. If private: hash password with bcrypt
3. Save RoomMetadata to MongoDB
4. Return { roomId, room: metadata }

GET /api/rooms/feed

// Query params: ?language=en (optional filter)

// Logic:
1. Find all RoomMetadata where:
   - published = true
   - gameState = "NOT_STARTED"
2. If language param: filter by language
3. Populate owner's displayName + avatar
4. Return { rooms: [...] }
Key design: Only rooms that haven't started yet appear in the feed. Once a game begins, the room disappears from the feed automatically.

POST /api/rooms/:roomId/publish

// Headers: Authorization: Bearer <jwt>

// Logic:
1. Find room by roomId
2. Verify requester is the owner (ownerId matches userId)
3. Set published = true
4. Return { success: true }

POST /api/rooms/:roomId/unpublish

// Same as publish but sets published = false

Profile Routes

server/src/routes/profile.ts

// No auth required

// Logic:
1. Find user by userId
2. Return { user: { displayName, avatar, stats } }
   (no email or password exposed)

PUT /api/profile

// Headers: Authorization: Bearer <jwt>
// Request body:
{ displayName?: string, avatar?: { body, eyes, mouth } }

// Logic:
1. authMiddleware extracts userId
2. Update only provided fields
3. Return { user: updated user }

Stats Routes

server/src/routes/stats.ts

POST /api/stats/update

// Headers: Authorization: Bearer <jwt>
// Request body:
{
  gamesPlayed: number,
  gamesWon: number,
  totalScore: number,
  drawingRounds: number,
  wordsGuessed: number
}

// Logic:
1. authMiddleware extracts userId
2. Use MongoDB $inc operator to atomically increment each stat
3. Return { success: true }
Why $inc? The client sends the delta (how much to increment), not the new total. This prevents race conditions if two game-end events arrive simultaneously.

Quiz: Test Your Understanding

What status code is returned when a duplicate email is used to register?

  • 400
  • 409
  • 422
  • 500

What does GET /api/rooms/feed return?

  • All rooms ever created
  • All private rooms
  • Published rooms that haven't started
  • Rooms with open slots only

Why does POST /api/stats/update use MongoDB's $inc operator?

  • It's faster than find-and-replace
  • It prevents race conditions on concurrent updates
  • It validates the input automatically
  • It logs changes to a audit trail
