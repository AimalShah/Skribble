Complete Game Walkthrough

Let's follow Alice (who opens the app) and Bob (who joins via invite link) through a full game.

Phase 1: Connection & Room Join

StepWhat HappensCode Path
1 Alice opens the app. Browser loads React, renders <JoinGameForm /> App.tsx → JoinGameForm.tsx
2 Alice enters name "Alice", selects language, customizes character, clicks "Play!" handleJoin(false) → socket.connect() → socket.emit("joinRoom", playerData, language)
3 Server receives joinRoom. No roomId provided → calls handleNewRoom() socketHandlers.ts:30-51
4 handleNewRoom → getPublicRoom(language) scans Redis for rooms with space. None found → generateEmptyRoom() creates a new room with UUID roomController.ts:282-301 → gameController.ts:26-53
5 handleNewPlayerJoin() creates a Player object with Alice's data + playerId = socket.id, pushes to room.players, saves to Redis. Alice's socket joins the Socket.IO room. roomController.ts:451-496
6 Server emits joinedRoom (to Alice) and playerJoined (to room). roomController.ts:481-482
7 Client's App.tsx receives joinedRoom, calls setRoom(room) → React re-renders showing <Game> instead of <JoinGameForm> App.tsx:22
8 Simultaneously, Alice's RoomContext receives joinedRoom → stores room, identifies "me" = Alice RoomContext.tsx:131-133

Phase 2: Private Room Setup (optional)

If Alice chose "Create Private Room":

  • isPrivate = true in step 2, generateEmptyRoom(socket, true) sets the creator = Alice's socket ID
  • Alice sees GameSettings — can change players, drawTime, rounds, etc.
  • Alice copies invite link (RoomLink) and shares it with Bob
  • Bob opens the link → React reads ?roomId=xxx from URL → handleJoin() emits joinRoom with Bob's data + roomId
  • Server matches Bob to Alice's room via handleNewPlayerJoin()

Phase 3: Starting the Game

StepWhat HappensCode Path
9 Alice (host) clicks Start. socket.emit("startGame", { words: [...] }) GameSettings.tsx:74-78
10 Server validates: Alice is creator? Game not started? 2+ players? socketHandlers.ts:55-69
11 startGame() resets game state to round 1, player 0. Emits gameStarted to all. roomController.ts:48-60
12 Client receives gameStarted. RoomContext sets myTurn for the drawer, stores room. RoomContext.tsx:147-152
13 nextRound() picks 3 random words, sends chooseWord to drawer ONLY, choosingWord to others. roomController.ts:150-188

Phase 4: Word Selection

StepWhat HappensCode Path
14 Drawer (Alice) sees WordSelector with 3 word buttons. 30s timer starts. OverlayContent.tsx → WordSelector.tsx
15 Bob sees "Alice is choosing a word" overlay with countdown. ChoosingWord.tsx
16 Alice clicks "elephant". socket.emit("wordSelect", "elephant") WordSelector.tsx:6-8
17 Server clears word-choose timer, sets roomState = DRAWING, emits wordChosen to drawer with the word, guessWordChosen to others with underscores. roomController.ts:191-230
18 If Alice doesn't pick in 30s, timeout auto-selects a random word. roomController.ts:180-187

Phase 5: Drawing & Guessing

StepWhat HappensCode Path
19 Alice sees the canvas + toolbar + the word "elephant". 60s timer starts. GameCanvas.tsx + Header.tsx
20 Bob sees blank canvas + underscores "_ _ _ _ _ _ _ _". 60s timer starts. Header.tsx renders renderWord() with underscores
21 Alice draws. Each mousemove emits "draw" with { x, y, color, lineWidth, end }. GameCanvas.tsx:53-69
22 Server relays each drawData to all other players. They call revieveDrawData(). roomController.ts:304-334 → GameCanvas.tsx:83-96
23 Bob types "elephant" and presses Enter. socket.emit("guess", { guess: "elephant" }). Chat.tsx:21-26
24 Server checks: Bob is not drawer? Word matches? Not already guessed? Yes → marks Bob as guessed. roomController.ts:111-148
25 Server emits guessed to all. Client shows "Bob has guessed the word" in green. Message.tsx:48-55
26 If everyone guessed (or timer expires), endRound() is called. roomController.ts:138-143 or roomController.ts:219-221

Phase 6: Hints (if enabled)

StepWhat Happens
27At 50% of drawTime (e.g., 30s into a 60s round), sendHint() fires.
28Server picks a random unrevealed character index, emits guessHint with { index, letter }.
29Bob's Header receives the hint, re-renders with the revealed letter animated via Framer Motion.
30After 10 more seconds, another hint is sent (up to settings.hints total).

Phase 7: Round End & Scoring

StepWhat Happens
31endRound() calculates scores: Bob gets max(200 - secondsToGuess, 0). Alice gets 50 + bonusPerGuesser.
32Server resets guessed flags, clears drawingData.
33Server emits turnEnded with the word. Both players see "The word was 'elephant'" for 5 seconds.
34After 5s pause, nextRound() is called. Current player advances to the next player.

Phase 8: Game End

StepWhat Happens
35After all rounds complete, endGame() emits gameEnded.
36Client shows Winners overlay with final scores sorted descending.
37If public room, after 10 seconds the game auto-restarts (startGame() called again).
38If private room, back to lobby (GameSettings).

Edge Cases & What Happens

ScenarioWhat the Server Does
Player disconnects mid-roundhandlePlayerLeft() → if they're the drawer, endRound(LEFT). Removes player from array. If only 1 left, endGame(). If empty room, deleteRedisRoom(). Passes creator torch.
Player joins mid-gamehandleInBetweenJoin() → sends GAME_STATE with current game state (without the word — underscores instead), remaining time, and hints so far.
Host leaves private roomCreator reassigned to room.players[0] — the next player becomes host.
Drawer doesn't pick a word30s timeout auto-selects a random word from the 3 options.
All guessers guess before timerendRound(ALL_GUESSED) — round ends early, saving remaining draw time.
Only 2 players, one disconnectsendRound() with LEFT reason checks players.length === 2 and returns early (doesn't end round). Then handlePlayerLeft sees only 1 player left and ends the game.

Known Bugs (From Code Inspection)

  • Undo for non-drawers: The UNDO_DRAW event triggers clearCanvas on receivers, but doesn't re-load the drawing data. Non-drawers see a blank canvas after undo.
  • MessagesContext cleanup: The cleanup useEffect uses socket.on() instead of socket.off() for some events (GAME_STARTED, KICKING_VOTE), creating duplicate listeners in StrictMode.
  • Timer interval leak: Header.tsx sets up setInterval but doesn't clean it up in the useEffect return.
  • Hardcoded server IP: client/src/socketHandler.ts has 192.168.29.63:8000 hardcoded — won't work on other networks.

💡 Interview Question: "Walk me through what happens when I type a guess and press Enter."

Client emits "guess" → server guessWord() validates → if correct, marks guessed=true, emits "guessed" → if all guessed, endRound() → if wrong, emits "guess" back (lowercase 'g') → MessagesContext receives it → adds to messages array → Chat re-renders.
