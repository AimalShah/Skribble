Client Contexts

The three React Context providers that manage all client-side state.

What You'll Learn

By the end of this lesson, you'll understand how AuthContext, RoomContext, and MessagesContext manage state and handle socket events on the client.

The Three Layers

<AuthProvider> ← Global auth state (user, token) <RoomProvider> ← Game state (room, players, turn) <MessagesProvider> ← Chat messages <Game /> ← Components consume all three </MessagesProvider> </RoomProvider> </AuthProvider>

AuthContext

client/src/context/AuthContext.tsx

AuthState {
  user: User | null;
  token: string | null;
  loading: boolean;
}

Functions:
  login(email, password)
    → POST /api/auth/login
    → Stores token + user in localStorage
    → Updates context state

  register(email, password, displayName)
    → POST /api/auth/register
    → Stores token + user in localStorage
    → Updates context state

  logout()
    → Clears localStorage
    → Sets user=null, token=null

  updateUser(user)
    → Updates context + localStorage

On mount:
  → If token exists in localStorage, calls GET /api/auth/me
  → Validates token is still good
  → If invalid: clears localStorage
Persistence: AuthContext persists the JWT token and user object to localStorage. On page reload, it re-validates the token with /me before showing the app.

RoomContext

client/src/context/RoomContext.tsx

RoomState {
  room: Room | null;        // Full room object
  myTurn: boolean;          // Is it my turn to draw?
  me: Player | null;        // My player object
  roomState: RoomState;     // Current game state
  mutedPlayers: Set<string>; // Player IDs I've muted
}

Socket event listeners:
  JOINED_ROOM     → Sets room, identifies "me" by socket.id
  WORD_CHOSEN     → Sets DRAWING state
  TURN_END        → Sets GUESSED state
  GAME_STARTED    → Initializes turn/round tracking
  GAME_ENDED      → Sets WINNER state (resets after delay)
  PLAYER_JOINED   → Adds player to room.players
  PLAYER_LEFT     → Removes player from room.players
  CHOOSE_WORD     → Sets PLAYER_CHOOSE_WORD state
  GUESS_WORD_CHOSEN → Sets DRAWING state
  CHOOSING_WORD   → Sets CHOOSING_WORD state
  GAME_STATE      → Updates gameState for mid-game joins

Functions:
  changeSetting(setting, value)
    → Updates local settings state

  mutePlayer(playerId)
    → Adds to mutedPlayers set

  removeMute(playerId)
    → Removes from mutedPlayers set

MessagesContext

client/src/context/MessagesContext.tsx

Message {
  sender?: string;
  message: string;
  type: MessageType;
}

MessageType enum:
  Guess | PlayerLeft | PlayerJoin | Error |
  WordGuessed | GuessClose | WordChoosen |
  WordWas | VoteKick

Socket event listeners:
  GAME_STARTED     → Clears all messages
  GUESS            → Adds wrong guess to chat
  GUESSED          → Adds "guessed correctly" message
  PLAYER_JOINED    → Adds join message
  PLAYER_LEFT      → Adds leave message
  WORD_CHOSEN      → Adds "drawer is drawing" message
  TURN_END         → Adds "word was: X" message
  KICKING_VOTE     → Adds vote kick message
  error            → Adds error message

addMessageToChat(msg):
  → Filters muted players
  → For drawer: shows "close" guesses differently
  → Adds to messages array
Close guess filtering: The drawer sees "close" guesses (wrong but similar) highlighted differently. This is server-side logic — the server sends a GUESS_CLOSE event when a guess is within edit distance of the word.
