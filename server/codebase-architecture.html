Skribble (Sync Draw Guess)

📅 Generated: 2026-06-18 📏 Lines of code: ~3,200 📁 Files: ~49 🔤 Language: TypeScript / React / Node.js

A real-time, multiplayer drawing-and-guessing game. Players join rooms, take turns drawing prompts, and compete to guess the drawings. Built with Socket.IO for real-time communication, React for the frontend, Express for the backend, and Redis for state persistence.

📋 Project Overview

What it does

Skribble is a web-based party game where one player draws a word while other players guess what it is. Each correct guess earns points; the drawer earns points for each successful guess. A round timer, hint system, and word-selection phase structure each turn. Rooms can be public (auto-matchmaking) or private (invite-only via shareable link).

Ecosystems

The project is split into two independent Node.js packages under /home/aimalshah/code/Skribble/client and /home/aimalshah/code/Skribble/server, both written in TypeScript.

🚪 Entry Points

How the application starts — from process launch to first meaningful code.

Server: server/src/server.ts:1-22

The server entry point creates an Express app, attaches CORS middleware, mounts Socket.IO on an HTTP server, then calls setupSocket(io) and setupCommandLine(io). It listens on port 8000.

  • Express HTTP server for potential REST endpoints (currently none defined)
  • Socket.IO configured with wildcard CORS (origin: "*")
  • Command-line interface for server admin commands (broadcast, stats, clear, exit)
const app = express();
const server = http.createServer(app);
app.use(cors());
const io = require("socket.io")(server, { cors: { origin: "*" } });
setupSocket(io);
setupCommandLine(io);
server.listen(8000);

Client: client/src/main.tsx:1-10

The client entry point mounts the App component into the #root DOM element using React 18's createRoot API, wrapped in StrictMode.

createRoot(document.getElementById('root')!).render(
  <StrictMode><App /></StrictMode>
);

Client: client/src/App.tsx:1-40

The root Home component manages the top-level room state. On mount, it listens for joinedRoom and kicked events. It conditionally renders either JoinGameForm (no room) or Game (in a room), wrapped in RoomProvider context.

Client: client/src/socketHandler.ts:1-5

Creates the singleton Socket.IO client instance pointing to http://192.168.29.63:8000 with autoConnect: false.

State Persistence

All room and game state is stored in Redis. The server reads/writes via getRedisRoom and setRedisRoom. Keys are prefixed with room: (private) or publicRoom: (public). State is serialized as JSON strings.
